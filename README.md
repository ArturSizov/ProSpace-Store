# ProSpace-Store

REST API для платформы электронной коммерции (интернет-магазина) с развитой системой разграничения прав доступа.

## 🛠 Технологический стек
* **Платформа:** .NET 8 / .NET 9, ASP.NET Core Web API
* **База данных & ORM:** PostgreSQL, Entity Framework Core (Code First)
* **Безопасность:** JWT Bearer Authentication, Ролевая модель авторизации (Identity / Custom Roles)
* **Архитектура:** Многослойная архитектура (API, Domain, DataAccess) с разделением ответственности

## 🚀 Реализованный функционал
* **Аутентификация и авторизация:** Полноценная регистрация пользователей, выдача JWT-токенов, разделение прав доступа (Пользователь / Менеджер / Администратор).
* **Каталог товаров:** CRUD-операции для управления продуктами и категориями.
* **Управление заказами:** Логика оформления заказов и изменения их статусов в зависимости от роли.

## 🧱 Архитектура проекта
Проект спроектирован по принципам Separation of Concerns (Разделение обязанностей):
* `ProSpace.Domain` — корпоративные бизнес-правила, сущности (Entities) и интерфейсы.
* `ProSpace.DataAccess` — инфраструктурный слой, контекст БД (DbContext), миграции и реализация репозиториев.
* `ProSpace.Api` — контроллеры, обработка HTTP-запросов, middleware для обработки ошибок.

## ⚡ Как запустить проект локально
1. Склонируйте репозиторий: `git clone ...`
2. Настройте строку подключения к PostgreSQL в `appsettings.json`.
3. Примените миграции: `dotnet ef database update --project ProSpace.DataAccess`
4. Запустите API: `dotnet run --project ProSpace.Api`
