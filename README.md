# ProSpace-Store

REST API для платформы электронной коммерции (интернет-магазина) с развитой системой разграничения прав доступа. Проект спроектирован по принципам Clean Architecture / DDD, полностью контейнеризирован и переведен на СУБД PostgreSQL для быстрого развертывания в production-окружении.

## 🛠 Технологический стек
* **Платформа:** .NET 8 / ASP.NET Core Web API
* **База данных & ORM:** PostgreSQL, Entity Framework Core (Code First)
* **Контейнеризация:** Docker, Docker Compose
* **Безопасность:** JWT Bearer Authentication, Ролевая модель авторизации (Identity / Custom Roles)
* **Документация:** Swagger UI / OpenAPI

## 🚀 Реализованный функционал
* **Аутентификация и авторизация:** Полноценная регистрация пользователей, выдача JWT-токенов, разделение прав доступа на уровне политик и ролей (`Пользователь` / `Менеджер` / `Администратор`).
* **Каталог товаров:** CRUD-операции для гибкого управления продуктами, категориями и остатками.
* **Управление заказами:** Бизнес-логика оформления покупок и изменения их статусов в зависимости от роли участника.

## 🧱 Архитектура проекта
Проект спроектирован по принципам **Separation of Concerns (Разделение обязанностей)**, обеспечивая независимость бизнес-логики от внешних фреймворков и СУБД:
* **`ProSpace.Domain`** — доменное ядро системы. Содержит корпоративные бизнес-правила, сущности (Entities), доменную логику магазина и интерфейсы репозиториев.
* **`ProSpace.Application`** — слой бизнес-сценариев. Координирует выполнение Use Cases и управляет потоками данных.
* **`ProSpace.DataAccess`** — инфраструктурный слой. Реализует паттерн Repository, содержит контекст БД (`ProSpaceDbContext`), конфигурации Fluent API и механизмы работы с PostgreSQL.
* **`ProSpace.Api`** — презентационный слой. Точка входа в приложение (REST API). Отвечает за контроллеры, JWT-валидацию, кастомные Middleware обработки ошибок и настройку DI-контейнера.

## ⚡ Инструкция по запуску

Проект полностью подготовлен для быстрого локального тестирования. Вам **не нужно** вручную устанавливать сервер PostgreSQL или настраивать .NET SDK — всё окружение автоматически разворачивается в изолированных контейнерах одной командой.

### Предварительные требования
На вашем компьютере должен быть установлен и запущен **Docker Desktop**.

### Запуск через Docker (Рекомендуемый способ)

1. **Клонируйте репозиторий** и перейдите в корневую папку:
   ```bash
   git clone https://github.com
   cd ProSpace-Store
   ```

2. **Запустите контейнеры** одной командой (из папки, где находится файл `docker-compose.yml`):
   ```bash
   docker-compose up --build
   ```

3. **Дождитесь инициализации**: Docker автоматически скомпилирует исходный код API и поднимет сервер PostgreSQL. При первом старте приложение самостоятельно создаст базу данных `prospace_store_db` и применит все необходимые таблицы.

### Развернутая инфраструктура:
* **`prospace-web-api`** (Порт `8080`) — контейнер с вашим ASP.NET Core приложением.
* **`prospace-postgres-db`** (Порт `5432`) — контейнер СУБД PostgreSQL. Данные сохраняются в Docker Volume `postgres_data`, поэтому они не удаляются при перезапуске контейнеров.

## 🔑 Тестовые учетные записи (Credentials)

В базу данных при старте приложения автоматически добавлены тестовые пользователи с разными уровнями доступа для проверки ролевой модели. Вы можете использовать их для генерации JWT-токенов:

### 💼 Роль: Менеджер (Manager)
* **Email:** `manager@example.com`
* **Пароль:** `!P@$$w0rd1`
* *Имеет доступ к управлению каталогом, изменению статусов заказов и модерации.*

### 👤 Роль: Обычный пользователь (Customer)
* **Email:** `johndoe@example.com`
* **Пароль:** `?KA{vrejW3^0=1Trrq`
* *Имеет доступ к просмотру товаров, созданию корзины и оформлению личных заказов.*
